My name is Simon and I make websites. Kinda. It's a bit more complicated than that – the things I make tend to be a fair bit more involved than just words on a screen – but that's what I tell people when I don't want to bore them too much.

Professionally I tend to work on data-heavy frontends, often producing large scale visualisations or professional quality CRUD interfaces. That stuff tends to be both copyright and slightly dull though so on this site I'm putting the fun bits I work on in my free time.

Welcome!

Most recently I revisited some CSS animation as it had been a while since I'd refreshed my knowledge. Turns out that with the advent of CSS properties and the implementation of more and more CSS features (different gradients and transforms and filters and whatnot) you can create effects today that would have been impossible with CSS alone 10 years ago.

I actually made this modal open animation after seeing a "how can this be done with CSS?" question on Reddit. It piqued my interest enough that I had a crack and got pretty damn close (if I do say so myself). While fiddling with that animation I got a few ideas for other ways to take it and so made this demo of different modal opening animations to try some of them out. The result was a number of very simple to code CSS animations that could be adapted and combined to almost any aesthetic, and I'm genuinely excited to slip a few in to my future work projects.

A while back I spent a good bit of time learning ThreeJS and WebGL. I think it really got me interested when I saw this demo of flocking birds, which I dissected until I could produce this box full of butterflies. The same approach was then streamlined from 3D into 2D and combined with some other fun techniques (such as generating a flow field based on the HTML) to produce the "ones and zeroes" effect you're seeing on this page.

I also half built a couple of game demo type things with ThreeJS. The first is a sort of cave-based maze/exploraton deal, using a rhombic dodecahedral lattice to prodcedurally generate the environment. The second is a more open asteroid/space vibe, which I think looks really cool even if it's kinda unfinished. There's a lot of shader magic in both of those but particularly the asteroid one, as well as things like game chunk loading and seed based "terrain" generation (at least the begginings of). The problem I've had with both concepts is that while I can make something that looks like a game it is a very different proposition to make something that is also fun/engaging to play. I've parked both projects until I can come up with a compelling use case (and then I'll probably just strip 'em for parts).
